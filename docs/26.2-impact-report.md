## 2  Movement Physics ÔÇö Complete Formula Rewrite

### 2a  Friction + Air Drag (LivingEntity.travelInAir)

New helper used everywhere:

```java
computeModifiedFriction(float friction, float modifier) =
    clamp(1.0 - (1.0 - friction) * modifier, 0.0, 1.0)
```

**Old formula:**
```text
blockFriction  = onGround ? block.getFriction() : 1.0
airDrag        = 0.91  (hardcoded)
friction       = blockFriction * 0.91
vertDrag       = isFlyingAnimal ? friction : 0.98
```

**New formula:**
```text
blockFriction  = onGround ? computeModifiedFriction(block.getFriction(), FRICTION_MODIFIER) : 1.0
airDrag        = computeModifiedFriction(0.91, AIR_DRAG_MODIFIER)
friction       = blockFriction * airDrag
vertDrag       = omnidirectionalAirMover() ? friction : computeModifiedFriction(0.98, AIR_DRAG_MODIFIER)
```

Constants extracted:
- `BASE_AIR_DRAG = 0.91f`
- `BASE_VERTICAL_DRAG_FOR_NON_FLYERS = 0.98f`
- `LIQUID_FLOAT_IMPULSE = 0.04f` (renamed from `WATER_FLOAT_IMPULSE`)

`FlyingAnimal` interface is no longer the physics switch ÔÇö replaced by `omnidirectionalAirMover()` virtual method.

### 2b  Bounce/Restitution System (Entity)

Old system: each block overrode `updateEntityMovementAfterFallOn()` (e.g. `SlimeBlock.bounceUp`).

New system: **centralized `restituteMovementAfterCollisions()`** on `Entity`:
- Called when `canSimulateMovement() && (verticalCollision || horizontalCollision)`
- Computes `restitution = getEntityBounciness()` (from attribute for LivingEntity, 0.0 at Entity level)
- Horizontal: reflects velocity `Ôłĺmovement.x * restitution`
- Vertical: uses `max(entityBounce, blockBounce)`, suppressed if velocity < gravity or block is `#suppresses_bounce`
- Gravity compensation: `(movement.y / currentMovement.y) * effectiveGravity`
- Fires `GameEvent.BOUNCE` when non-zero bounce occurs

New Entity methods:
- `getBlockBounciness(Block)` Ôćĺ `block.getBounceRestitution()` (├Ś0.8 for non-LivingEntity)
- `getEntityBounciness()` Ôćĺ attribute value on LivingEntity
- `getEffectiveGravity()`
- `omnidirectionalAirMover()` Ôćĺ false (overridden by SulfurCube to true when it has a body item)

### ReaperAC action
- Port friction formula to use `computeModifiedFriction` with FRICTION_MODIFIER/AIR_DRAG_MODIFIER.
- Implement bounce restitution in the velocity predictor.
- Track `#suppresses_bounce` tag for vertical bounce suppression.
- Replace `FlyingAnimal` checks with `omnidirectionalAirMover()` equivalent.

### Verification

- Add `travelInAir()` parity tests comparing old/new friction + vertical drag outcomes, including the `omnidirectionalAirMover()` branch.
